# AltTab — working notes for Claude

Minimal, fast macOS window switcher (Swift / AppKit, SwiftPM executable bundled into a `.app`).
Overrides native Cmd+Tab. Full rationale: `docs/DECISIONS.md`. Environment: macOS 26.x, Swift 6.3.x,
Xcode toolchain, Apple Silicon.

## Building & signing — READ THIS FIRST

**Always build with `./build.sh`.** It signs with the stable identity pinned in `.signing-identity`
(gitignored, machine-local). Do **NOT** produce an ad-hoc build for the user to run.

Why it matters: the app needs the macOS **Accessibility (TCC)** permission. TCC keys the grant on the
code-signing *designated requirement*. A **stable** identity (Apple Development / Developer ID /
self-signed) ⇒ stable requirement ⇒ the grant **survives rebuilds**. **Ad-hoc** signing (`SIGN_IDENTITY=-`,
the bare-`./build.sh` default *without* `.signing-identity`) changes the code hash every build ⇒ the grant
is **dropped on every rebuild** and the user must re-grant. This already bit us once and littered TCC with
stale entries. `build.sh` reads `.signing-identity` so a plain `./build.sh` is correctly signed — keep that
file in place; if it's missing, recreate it (the identity is in the user's keychain:
`security find-identity -v -p codesigning`).

## Runtime / deployment

- The user runs **`/Applications/AltTab.app`** (they move/replace `build/AltTab.app` there themselves).
  Because the DR is cert-based, replacing the bundle with a freshly *stably-signed* build keeps the grant.
- It's an `LSUIElement` agent (menu-bar only, no Dock tile). Bundle id: `dev.fusy.alttab`.
- App icon: `Resources/AltTab.icns`, generated by `scripts/makeicon.swift` (CoreGraphics, no asset catalog).

## If the Accessibility grant "resets" / native Cmd+Tab comes back

Symptom: the default Apple Cmd+Tab appears = AltTab isn't disabling the native switcher = it lacks
Accessibility (the whole stack is gated behind it). Checklist:
1. Confirm the **running** bundle is the stably-signed one: `codesign -dv --verbose=4 <app>` (look for a
   real Authority, not "adhoc") and `codesign -d -r- <app>` (DR should reference the cert leaf, not a cdhash).
2. Clear stale/conflicting TCC entries, then re-grant **once**:
   `tccutil reset Accessibility dev.fusy.alttab`
3. Verify objectively: AltTab disables symbolic hotkeys 1 (Cmd+Tab) & 2 (Cmd+Shift+Tab) when active —
   probe via `CGSIsSymbolicHotKeyEnabled`/`SLSIsSymbolicHotKeyEnabled` (1 = native ON / AltTab not
   overriding; 0 = AltTab active).

## Code map (quick)

`HotkeyManager` (Carbon hotkey + CGEventTap, disables native Cmd+Tab) · `SwitcherController` (session state
machine, fast-flip) · `Switcher/*` (native-style panel: one centered icon row + selected-title label;
**single-row invariant** — `chooseIconSize` shrinks icons by *width* so all windows stay on ONE row, never
wraps; assumes count ≤ what fits at `minIcon`) ·
`Model/WindowStore` + `AppObserver` + `AXSupport` (AX-driven window model, per-window MRU; all AX IPC off
the main thread via the serial `AXQueue`, model mutated on main) · `Focus` + `PrivateAPIs` (SLPS focus path,
the only private symbols). Swift 5 language mode on purpose (background run-loop threads for AX/event tap).
